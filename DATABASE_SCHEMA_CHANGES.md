# 数据库结构和功能修改总结

## 📋 修改概览

本次修改对文章管理系统进行了重大重构，分离了章节基本信息生成和章节内容生成的流程，新增了伏笔管理系统。

## 🗄️ 数据库表结构修改

### 1. 文章表 (article) 修改
**新增字段：**
- `total_word_count_estimate INTEGER` - 总字数预估
- `chapter_word_count_estimate INTEGER` - 每章节字数预估

### 2. 章节表 (article_chapter) 修改
**新增字段：**
- `core_plot TEXT` - 核心剧情
- `word_count_estimate INTEGER` - 字数预估

### 3. 伏笔表 (plot) 新建
**字段结构：**
- `id BIGSERIAL PRIMARY KEY` - 主键ID
- `article_id BIGINT NOT NULL` - 文章ID
- `chapter_id BIGINT NOT NULL` - 埋设伏笔的章节ID
- `plot_name VARCHAR(255) NOT NULL` - 伏笔名称
- `plot_content TEXT` - 伏笔内容
- `recovery_chapter_id BIGINT` - 回收伏笔的章节ID
- `res_state SMALLINT DEFAULT 1` - 删除标记
- `create_time TIMESTAMP` - 创建时间
- `update_time TIMESTAMP` - 更新时间

## 🤖 定时任务重构

### 1. ArticleChapterGenerationTask 修改
**原有功能：** 生成章节内容
**修改后功能：**
- 只生成章节基本信息（章节名称、字数预估、核心剧情、伏笔列表）
- 根据文章的总字数预估和每章节字数预估计算章节数量
- 调用AI生成所有章节的基本信息
- 保存章节信息和伏笔信息到数据库

**执行频率：** 每10分钟一次

### 2. ArticleContentGenerationTask 新建
**功能：** 根据章节基本信息生成章节具体内容
**核心逻辑：**
- 查询没有内容的章节（chapter_content为空）
- 为每个章节构建详细的AI提示词
- 包含伏笔埋设规则和回收计划
- 生成符合字数要求的章节内容
- 确保结尾留下强悬念

**执行频率：** 每15分钟一次

## 🔧 代码结构修改

### PO/DTO/BO类更新
- `Article.java` - 添加字数预估字段
- `ArticleChapter.java` - 添加核心剧情和字数预估字段
- `Plot.java` - 新建伏笔实体类
- 相应的DTO类都已更新

### Mapper接口和XML更新
- `ArticleMapper` - 添加字数预估字段映射
- `ArticleChapterMapper` - 添加新字段映射和查询方法
- `PlotMapper` - 新建完整的CRUD操作
- 所有XML文件都已更新相应的SQL语句

## 🎯 AI提示词优化

### 章节基本信息生成提示词
包含：
- 文章基本信息和大纲
- 生成配置（性别、题材、情节等）
- 预计章节数和字数要求
- 输出格式要求（JSON）

### 章节内容生成提示词
包含四个主要部分：
1. **基础信息** - 标题、核心剧情、字数预估
2. **伏笔规则** - 严格的埋设要求
3. **伏笔回收计划** - 明确的回收时间表
4. **写作要求** - 字数控制、节奏要求、结尾悬念

## 📊 工作流程优化

### 原有流程
文章创建 → 直接生成章节内容 → 完成

### 新优化流程
1. **文章创建** - 设置总字数和每章字数预估
2. **章节基本信息生成** - 生成所有章节的名称、剧情、字数、伏笔
3. **章节内容生成** - 根据基本信息生成具体内容，严格遵循伏笔规则

### 优势
- **更好的内容控制** - 可以预先规划整篇文章的结构
- **伏笔管理** - 系统化管理伏笔的埋设和回收
- **分阶段生成** - 避免一次性生成大量内容导致质量下降
- **可维护性** - 每个阶段职责清晰，便于调试和优化

## 🚀 使用说明

### 前提条件
文章必须设置：
- `total_word_count_estimate` - 总字数预估
- `chapter_word_count_estimate` - 每章节字数预估

### 执行顺序
1. 创建文章并设置字数预估
2. `ArticleChapterGenerationTask` 自动生成章节基本信息
3. `ArticleContentGenerationTask` 自动生成章节具体内容

### 监控要点
- 检查伏笔的埋设是否自然
- 验证章节字数是否符合预估
- 确认结尾悬念设置是否恰当
- 监控AI生成质量和时间消耗

## 📝 注意事项

1. **伏笔管理** - 伏笔的埋设和回收必须严格遵守计划
2. **字数控制** - AI生成时会严格按照预估字数要求
3. **内容质量** - 分阶段生成可以提高整体内容质量
4. **数据库迁移** - 现有数据会自动适应新的表结构
